# Task: Workshop de Hooks Textuais

## Objetivo
Gerar uma bateria de hooks textuais altamente testáveis para anúncios, focados em parar o scroll e criar irresistibilidade de leitura nos primeiros 3 segundos.

## Inputs Necessários
- Produto / serviço
- Dor principal do público-alvo
- Maior desejo/transformação que o produto oferece
- 1-3 resultados concretos de clientes (se disponível)
- Plataforma alvo (define tamanho e estilo do hook)

## Entendendo o Hook Perfeito

### Anatomia
```
HOOK = [Elemento de parada] + [Relevância imediata] + [Tensão não resolvida]
```
O hook não vende. O hook faz a pessoa **querer continuar lendo**. A venda é trabalho do corpo do anúncio.

### Os 7 Gatilhos de Hook que Funcionam
1. **Curiosidade** — cria lacuna de informação que o cérebro precisa fechar
2. **Identificação** — "isso é sobre mim" → elimina o banner blindness
3. **Contraintuitivo** — viola expectativa → o cérebro para para processar
4. **Urgência/escassez** — ativa medo de perda
5. **Prova específica** — número real > qualquer afirmação genérica
6. **Problema nomeado** — nomear a dor é 50% da solução
7. **Promessa ousada** — promete transformação específica e crível

## Passo a Passo

### STEP 1: Mapear a Dor em 3 Níveis
```
Nível 1 — Superficial (o que eles reclamam):
"[Reclamação explícita do público]"

Nível 2 — Emocional (como os faz sentir):
"[Emoção negativa que a dor causa — vergonha, frustração, medo]"

Nível 3 — Identitário (o que diz sobre eles):
"[Crença limitante que a dor reforça]"
```

### STEP 3: Ranquear e Selecionar os TOP 5
Critério de seleção:
- Seria capaz de parar meu scroll se eu visse isso?
- Cria tensão que só se resolve lendo o resto?
- É específico o suficiente para não parecer genérico?
- Está alinhado com o nível de consciência do público?

### STEP 4: Para Cada Hook Selecionado — Escrever o Corpo
O corpo do anúncio deve:
1. Imediatamente expandir/validar a promessa do hook
2. Aprofundar a dor OU desenvolver a curiosidade
3. Apresentar a solução de forma natural
4. Terminar com CTA claro

### STEP 5: Preparar para Teste
Estruturar como testar:
```
TESTE A/B RECOMENDADO:
- Hook 1 vs Hook 2 (mesmo corpo)
- Mesmo público, mesmo criativo
- Período mínimo: 7 dias ou 1000 impressões
- Métrica de decisão: CTR + Taxa de Conversão
```

## Output Esperado
20 hooks organizados por categoria, top 5 ranqueados com justificativa, corpo de anúncio para cada um dos top 5 e protocolo de teste.
